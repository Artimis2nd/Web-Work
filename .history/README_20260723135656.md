## ฟีเจอร์

### 1. ระบบใบงาน (Log Group)
- การบันทึกข้อมูลหนึ่งครั้ง = 1 ใบงาน (มี GroupID เดียวกัน)
- 1 ใบงานประกอบด้วย: วันที่, ไซต์งาน, รายละเอียดงาน, ผู้สั่งงาน, รายชื่อคนงาน, รูปภาพ
- รองรับการแก้ไขใบงาน (Edit) — กดปุ่มแก้ไขจากแดชบอร์ด

### 2. การคำนวณค่าแรงแบบใหม่
- **รายชั่วโมง/OT**: เลือกชั่วโมงปกติ (1-8) และ OT (1-4) พร้อมกันได้
  - OT คิด 2 เท่าของค่าแรงรายชั่วโมง
  - มี label "4 ชม. (ครึ่งวัน)" และ "8 ชม. (เต็มวัน)"
- **เหมาจ่าย**: กรอกจำนวนเงินเอง — ไม่คิด +20%
- **คนงาน "เฟิร์น"**: ยกเว้น +20% ทุกกรณี (hardcode)

### 3. การแนบรูปภาพ
- อัปโหลดสูงสุด 6 รูปต่อ 1 ใบงาน
- บีบอัดรูปอัตโนมัติด้วย Canvas (resize 1000px, JPEG quality 0.7)
- ส่งเป็น Base64 → Backend สร้างไฟล์ใน Google Drive → เก็บ URL ในชีต Images (แยกจาก DailyLogs)
- รองรับ Drag & Drop

### 4. Autocomplete
- ฟิลด์ "ไซต์งาน" และ "ผู้สั่งงาน" มีประวัติการกรอกเก่าให้เลือก
- ข้อมูลมาจากค่าที่เคยบันทึกใน Sheets

### 5. แดชบอร์ดแบบใหม่
- แสดงรายการใบงาน (ไม่ใช่รายคน)
- แยกยอดค่าแรงปกติ+OT (รวม +20%) และค่าแรงเหมา (ไม่มี +20%)
- ปุ่มแก้ไขใบงานแต่ละรายการ
- เลือกใบงานหลายใบแล้วกดปุ่ม "พิมพ์สรุป" เพื่อเปิดหน้า summary-print.html

### 6. พิมพ์สรุปค่าแรง (Summary Print)
- เปิดหน้า `summary-print.html` ผ่าน `window.open()` จากแดชบอร์ด
- ข้อมูลถูกส่งผ่าน `sessionStorage` (ไม่ต้องเรียก API ซ้ำ)
- แสดงการ์ดแบบ A4 พร้อมรูปภาพ จัดเรียง 2 การ์ดต่อหน้า
- เหมาะสำหรับการพิมพ์เอกสารสรุปส่งให้ผู้เกี่ยวข้อง

## ขั้นตอนที่ 1: ตั้งค่า Backend (Google Apps Script)

1. สร้าง Google Sheets ใหม่ 1 ไฟล์ (จะเป็นฐานข้อมูลของระบบ) — ตั้งชื่อ เช่น `WageSystem-Data`
2. เปิดเมนู **Extensions > Apps Script**
3. ลบโค้ดเดิมทั้งหมดในไฟล์ `Code.gs` แล้ววางโค้ดจากไฟล์ `Code.gs` ที่ให้มาแทน
4. เปิด `Code.gs` และกำหนดค่า `DRIVE_FOLDER_ID` (บรรทัดที่ 26) — สร้างโฟลเดอร์หลักใน Google Drive สำหรับเก็บรูปภาพ แล้วใส่ Folder ID
5. รันฟังก์ชัน `setupSheets` หนึ่งครั้ง (เลือกฟังก์ชันจาก dropdown ด้านบน แล้วกด Run)
   - ระบบจะขอ Authorize สิทธิ์เข้าถึง Google Sheets และ Google Drive ของบัญชีคุณ — กด "Allow"
   - ฟังก์ชันนี้จะสร้างชีต `Workers`, `DailyLogs` และ `Images` พร้อมหัวคอลัมน์ให้อัตโนมัติ
6. Deploy เป็น Web App:
   - เมนู **Deploy > New deployment**
   - เลือกประเภท (Select type) เป็น **Web app**
   - Description: ใส่ชื่ออะไรก็ได้ เช่น `wage-api-v2`
   - **Execute as**: `Me` (บัญชีของคุณ)
   - **Who has access**: `Anyone` (จำเป็น เพื่อให้ GitHub Pages เรียกเข้ามาได้โดยไม่ต้อง Login)
   - กด **Deploy** แล้วคัดลอก **Web app URL** ที่ได้ (จะลงท้ายด้วย `/exec`)
7. ทุกครั้งที่แก้ไขโค้ด `Code.gs` ในอนาคต ต้องไปที่ **Deploy > Manage deployments** กดไอคอนดินสอ (Edit) แล้วเลือก **New version** เพื่ออัปเดต URL เดิมให้ใช้โค้ดล่าสุด

> **หมายสำคัญ**: การอัปโหลดรูปภาพต้องใช้สิทธิ์ Google Drive — ต้อง authorize ตอนรัน `setupSheets()` หรือตอนเรียกใช้ครั้งแรก

## ขั้นตอนที่ 2: ตั้งค่า Frontend

1. เปิดไฟล์ `assets/js/config.js`
2. แก้ไขค่า `API_URL` ให้เป็น Web App URL ที่ได้จากขั้นตอนที่ 1:
   ```js
   window.API_URL = 'https://script.google.com/macros/s/xxxxxxxxxxxxxxxx/exec';
   ```
3. อัปโหลดไฟล์ทั้งหมด (ยกเว้น `Code.gs` และ `README.md`) ขึ้น GitHub Repository
4. เปิดใช้งาน GitHub Pages: **Settings > Pages > Branch: main / (root)**
5. เข้าใช้งานผ่าน URL ที่ GitHub Pages มอบให้ เช่น `https://yourname.github.io/Web-Work/`

## โครงสร้างข้อมูลใน Google Sheets

### ชีต `Workers`

| คอลัมน์ | ประเภท | คำอธิบาย |
|---|---|---|
| `ID` | ตัวเลข | รหัสคนงาน (ระบบสร้างให้อัตโนมัติ) |
| `FullName` | ข้อความ | ชื่อ-นามสกุลคนงาน |
| `DailyWage` | ตัวเลข | ค่าแรงขั้นต่ำต่อวัน (บาท) |
| `Status` | ข้อความ | `Active` หรือ `Inactive` |

### ชีต `DailyLogs`

| คอลัมน์ | ประเภท | คำอธิบาย |
|---|---|---|
| `ID` | ตัวเลข | รหัสรายการ (1 แถว = คนงาน 1 คน ในใบงาน 1 ใบ) |
| `GroupID` | ข้อความ | รหัสผูกกลุ่ม — ใช้เมื่อบันทึกงานครั้งเดียวมีคนงานหลายคน |
| `Date` | วันที่ | วันที่ทำงาน |
| `Site` | ข้อความ | ไซต์งาน / โครงการ |
| `JobDetail` | ข้อความ | รายละเอียดงาน |
| `RequestedBy` | ข้อความ | ผู้สั่งงาน |
| `WorkerName` | ข้อความ | ชื่อคนงาน |
| `DailyWage` | ตัวเลข | ค่าแรงต่อวันของคนงาน |
| `WageType` | ข้อความ | `hourly` หรือ `fixed` |
| `Hours` | ตัวเลข | จำนวนชั่วโมงปกติ (0-8) |
| `OTHours` | ตัวเลข | จำนวนชั่วโมง OT (0-4) |
| `FixedAmount` | ตัวเลข | จำนวนเงินเหมาจ่าย |
| `RawWage` | ตัวเลข | ค่าแรงดิบ (คำนวณจาก WageType + Hours/OT/Fixed) |
| `TotalWithMarkup` | ตัวเลข | ค่าแรงรวม +20% (×1.2 ยกเว้น "เฟิร์น" และเหมาจ่าย) |
| `CreatedAt` | วันที่-เวลา | เวลาที่บันทึกเข้าระบบ |

### ชีต `Images`

| คอลัมน์ | ประเภท | คำอธิบาย |
|---|---|---|
| `GroupID` | ข้อความ | รหัสใบงาน (ใช้อ้างอิงกับ DailyLogs) |
| `ImageUrls` | ข้อความ | JSON array ของ URLs รูปภาพ |

> หมายเหตุ: ภาพถ่ายถูกแยกเก็บในชีต `Images` แทนการเก็บใน `DailyLogs` เพื่อลดขนาดข้อมูลในแถวหลัก และรองรับการเพิ่ม/ลบรูปภาพหลายรูปได้สะดวก

## สรุป API Endpoints (Action-based ผ่าน POST เดียว)

Frontend เรียกทุก action ผ่าน `POST` ไปที่ URL เดียวกัน โดยส่ง body เป็น JSON รูปแบบ `{ "action": "...", "payload": {...} }`

| Action | คำอธิบาย |
|---|---|
| `getDashboard` | ข้อมูลสรุปสำหรับหน้าแดชบอร์ด (KPI + ใบงานล่าสุด) |
| `getWorkers` | รายชื่อคนงานทั้งหมด |
| `addWorker` / `updateWorker` / `deleteWorker` | จัดการคนงาน |
| `getLogs` | รายการบันทึกงานทั้งหมด |
| `addLogs` | บันทึกใบงานใหม่ (รับคนงานหลายคน + รูปภาพ) |
| `getLogGroup` | ดึงข้อมูลใบงานตาม GroupID (สำหรับแก้ไข) |
| `updateLogGroup` | อัปเดตใบงานที่มีอยู่ |
| `deleteLogGroup` | ลบใบงานทั้งชุด (ตาม GroupID) |
| `getReport` | สรุปยอดรายบุคคล (กรองตามชื่อ/ช่วงวันที่) |
| `getPivotReport` | สรุปยอดรวมตามวันหรือไซต์งาน |
| `getSiteHistory` | ดึงรายชื่อไซต์งานที่เคยบันทึก (สำหรับ autocomplete) |
| `getRequesterHistory` | ดึงรายชื่อผู้สั่งงานที่เคยบันทึก (สำหรับ autocomplete) |

## หมายเหตุทางเทคนิค

- Frontend เรียก API ด้วย `Content-Type: text/plain` โดยเจตนา (ดูใน `assets/js/api.js`) เพื่อให้ Browser ส่งเป็น "simple request" หลีกเลี่ยงปัญหา CORS preflight (OPTIONS) ที่ Google Apps Script ไม่รองรับ
- ทุกหน้ามี Loading Skeleton ระหว่างรอข้อมูล และ Error Banner พร้อมปุ่ม "ลองใหม่" เมื่อเชื่อมต่อ API ไม่สำเร็จ
- การบีบอัดรูปภาพทำที่ฝั่ง Frontend ด้วย Canvas (resize + JPEG compression) ก่อนส่ง Base64 ไปยัง Backend
- รูปภาพถูกเก็บใน Google Drive Folder (ตามค่า `DRIVE_FOLDER_ID` ใน `Code.gs`) และตั้งค่าเป็น Anyone with link can view
- ภายในโฟลเดอร์หลัก Backend จะสร้างโฟลเดอร์ย่อยตามวันที่: `{วันที่}_A{timestamp}_{GroupID4ตัวแรก}/`
- ข้อมูล URLs รูปภาพถูกบันทึกในชีต `Images` (แยกจาก `DailyLogs`) เพื่อลดขนาดข้อมูล
- ค่า +20% คำนวณที่ฝั่ง Backend (`Code.gs`) เพื่อป้องกันความไม่ตรงกันหากมีการแก้ไขสูตรในอนาคต
- คนงานชื่อ "เฟิร์น" ถูก hardcode ให้ยกเว้น +20% ทุกกรณี
